# Regression Checklist

## Purpose

この文書は、`mp/main.py` 周辺の整理や将来の拡張を進める前に、壊してはいけない基本動作を固定するための手動回帰チェックリストである。

主な用途は次の通り。

- リファクタリング前後で挙動差分が出ていないか確認する
- 新機能追加時に既存互換性を崩していないか確認する
- 不具合報告時に最低限の再現確認をそろえる

## Preconditions

確認前に、少なくとも次を満たしていること。

- 対象ファームウェアが Pico / Pico 2 に書き込まれている
- `mp/` 配下の必要ファイルが配置されている
- `roms/rom0.bin` と `roms/rom1.bin` が存在する
- LCD とタッチ、必要に応じて USB キーボードと SD カードが接続されている

## Test Scope

本チェックリストでは、次の基礎動作を対象とする。

- 起動
- ROM 読み込み
- 画面更新
- キー入力
- スリープ復帰
- save-state
- スクリーンショット取得
- SD カード利用

## Core Checklist

### 1. Basic Boot

- [ ] `main.py` で通常起動できる
- [ ] 起動時に致命的な例外が発生しない
- [ ] 初期画面が LCD に表示される
- [ ] ステータス表示や表示更新が止まらない

確認メモ:

- シリアルログに `PB-1000 Emulator Starting...` と初期化完了メッセージが出ること
- `PC=` 表示が極端に不正な値になっていないこと

### 2. ROM Load

- [ ] `rom0.bin` と `rom1.bin` を読み込める
- [ ] ROM 読み込み失敗時に異常終了ではなく警告で分かる
- [ ] 正常時は起動後に PB-1000 相当の待機状態へ入る

### 3. Display Refresh

- [ ] LCD 表示が定期更新される
- [ ] 表示が乱れたまま固定されない
- [ ] 起動直後から数十秒観察してもフレーム更新が止まらない

### 4. Keyboard Input

- [ ] 英数字入力が取り込まれる
- [ ] `!` に割り当てた `EXE` が動作する
- [ ] 特殊キー変換 (`MODE`, `CAL`, `LCKEY`, `BRK`) が効く
- [ ] キー押下後に押しっぱなしにならず解放される

確認対象:

- `KeyboardInputManager`
- キュー投入
- 押下開始
- 解放タイミング
- 入力ブロック時の抑止

### 5. USB Keyboard

- [ ] USB キーボード接続時に通常起動できる
- [ ] C キーボードルーティングが有効になる
- [ ] USB キー入力で基本操作ができる
- [ ] F11 で save-state 要求が入る
- [ ] F9 でリセット要求が入る

### 6. Touch Input

- [ ] タッチパネル接続時に通常起動できる
- [ ] LCD 領域内のタッチが `TK1` から `TK16` に変換される
- [ ] 指を離したあと押下状態が残らない

### 7. Sleep / Wake

- [ ] スリープ状態へ入ってもシステムが停止しない
- [ ] スリープ中に `BRK` 相当入力で復帰できる
- [ ] 復帰後に通常キー入力へ戻れる
- [ ] 復帰後に画面更新が継続する

### 8. Save State

- [ ] save-state 要求時に UI が更新される
- [ ] 保存成功時に成功メッセージが出る
- [ ] 保存失敗時に異常終了せずエラー表示になる
- [ ] USB キーボード利用時は保存前後で背景タイマ停止と再開が行われる

### 9. Capture

- [ ] PrintScreen 相当入力でキャプチャ処理が走る
- [ ] PBM ファイルが保存される
- [ ] VRAM ダンプが保存される
- [ ] 保存先が SD マウント有無で切り替わる

### 10. Shutdown / Exception Path

- [ ] `KeyboardInterrupt` で停止しても後処理に入る
- [ ] 例外終了時にワークエリアとメモリダンプ処理が動く
- [ ] 後処理で二次例外が起きにくい

## Suggested Smoke Scenarios

毎回すべてを確認できない場合は、最低限次の順で確認する。

1. 通常起動
2. 英数字入力と `EXE`
3. 画面更新継続
4. save-state
5. USB キーボードまたはタッチのどちらか一方

## Recommended Evidence

回帰確認時は、必要に応じて次を記録する。

- 使用したファームウェア識別情報
- 実行したエントリポイント
- 接続した周辺機器
- 成功 / 失敗したチェック項目
- 取得したログやスクリーンショットの保存先

## Current Gaps

現時点では自動化されていない確認が多いため、次の項目は将来の補強候補とする。

- `main.py` の起動スモークテスト
- save-state と capture の副作用確認
- USB キーボード経路の自動回帰
- スリープ復帰の再現テスト
