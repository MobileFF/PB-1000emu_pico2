# 開発ガイド

このガイドは、PB-1000 エミュレータの内部アーキテクチャを理解したい開発者、または開発に貢献したい方向けに作成されました。

## 1. システム・アーキテクチャ

このエミュレータは、パフォーマンスと柔軟性のバランスをとるために、C と Python のハイブリッド・アーキテクチャを採用しています。

- **CPU コア (C)**: HD61700 エミュレータコアは、最大の実行速度を実現するために C 言語で記述されています。命令のデコード、レジスタ管理、および基本的なタイミングを処理します。
- **MicroPython フレームワーク**: 高レベルのシステムロジック、周辺機器のエミュレーション (LCD、キーボード、SD カード)、およびメインの実行ループは、MicroPython で実装されています。
- **C-Module (ブリッジ)**: カスタムの MicroPython モジュール (`hd61700`) がブリッジとして機能し、CPU コントロールとメモリへのアクセスを Python に公開します。

## 2. ディレクトリ構造

- `src/`: C 言語ソースコード。
  - `hd61700.c / .h`: コア CPU エミュレーションロジック。
  - `modhd61700.c`: MicroPython モジュールラッパー。メモリマッピングと割り込みルーティングを処理します。
  - `lcd_controller.c / .h`: C 言語による高速 LCD レンダリングロジック。
- `mp/`: MicroPython ソースコード。
  - `main.py`: エントリポイント、システムを初期化し、メインループを実行します。
  - `pb1000.py`: `PB1000System` クラス、ボードレベルのエミュレーションを管理します。
  - `ili9341.py`: TFT ディスプレイ用の低レベル SPI ドライバ。
  - `keymap.py`: キーボード変換テーブル。
- `hardware/`: KiCad 回路図ファイル。

## 3. CPU コアの詳細

### メモリマッピング
`hd61700` モジュールは、2 つのメモリモードをサポートしています:
1.  **C-Managed (デフォルト)**: メモリバッファ (`rom0_buf`, `ram_buf` など) は C 言語で割り当てられます。これにより、CPU コアが Python の境界を越えることなく直接アクセスできるため、最高のパフォーマンスが得られます。
2.  **Python-Managed (デバッグ)**: メモリアクセスにより Python のコールバックがトリガーされます。特定のメモリサイクルをデバッグするのに便利ですが、大幅に低速になります。

### 周辺機器 I/O
- **ポート I/O**: HD61700 の `P0-P7` ポートは、`set_port_callbacks()` を介して Python のコールバックにマッピングされます。
- **LCD (HD61830)**: C サブモジュールとしてエミュレートされます。Python 側は、レンダリングのために物理的な SPI ディスプレイオブジェクトを C 側に提供します。
- **UART (MMIO)**: 0x0C00 範囲は C 言語でキャッチされます。内部 FIFO を使用しており、Python 側が物理的な PIO UART と通信するためにこれらを提供します。

## 4. 主な実装詳細

### PIO UART
Raspberry Pi Pico 2 にはハードウェア UART が限られている（そのうちの 1 つはコンソールに使用されている）ため、**Programmable I/O (PIO)** を使用して、4800 ボーのソフトウェアベースのビットバン UART を実装しています。これにより、タイミングに敏感なシリアル通信がメインの CPU エミュレーションをブロックしないようにしています。

### キーボード・マトリックス
PB-1000 は 13 行 x 12 列のマトリックスを使用しています。
- `keymap.py` で座標を定義します。
- USB ホストモードの場合、スキャンコードは `usb_host` モジュールによってキャプチャされ、C コアの `hd61700.press_row_ki()` に送信されます。

## 5. デバッグとトレース

### C 言語側のデバッグ
詳細な CPU トレースを有効にするには、`src/modhd61700.c` を編集します:
```c
#define DEBUG 1
```
または Python から呼び出します:
```python
import hd61700
hd61700.reset(True) # デバッグフラグを有効にする
```

### トレース・フック
C コアは、`PROG_TRACE_START` (0xB5D6) のような特定のウォッチポイントをサポートしており、`PBFTOBIN` やアセンブリローダーなどのプログラムのデバッグを支援します。

## 6. ビルドシステム

プロジェクトは MicroPython の `USER_C_MODULES` 機能を使用しています。`src/micropython.cmake` ファイルは、どの C ファイルをコンパイルし、どのようにリンクするかを MicroPython ビルドシステムに伝えます。
## 7. UTF-8 の運用ルール

このプロジェクトでは、テキストファイルの文字コードを UTF-8 に統一する。

- Markdown、Python、設定ファイルは UTF-8 を使用する
- Python ソースは UTF-8 BOM なしを推奨する
- Windows PowerShell から直接保存する場合は文字化けに注意する
- 文字コードを明示できる場合は常に `utf-8` を指定する

### 推奨設定

- VS Code の `files.encoding` は `utf8` にする
- `files.autoGuessEncoding` は必要がなければ無効にする
- 日本語を含む文書は、保存後に文字化けしていないことを確認する

### Python での推奨ファイル操作

```python
from pathlib import Path

text = Path("doc/example.md").read_text(encoding="utf-8")
Path("doc/example.md").write_text(text, encoding="utf-8")
```

### 運用上の注意

- PowerShell の `Set-Content` は環境によって BOM やコードページの影響を受けることがある
- 文字化けを避けるため、Codex からのファイル更新は Python の `read_text` / `write_text` を優先する
- 既存ファイルの再保存時は `utf-8-sig` で読み込み、`utf-8` で保存すると BOM 除去に使える
